### 2. 本次代码改动目标

目标：

```text
让 V11 主路径不再为了使用 NER 而初始化旧的 MedicalStandardizer。
```

背景：

```text
MedicalStandardizer 是旧的整句级 NER + 检索标准化编排。
当前 /expand/simple 主链路已经是 abbreviation record 级处理。
但 fallback 候选扩写后仍然需要 NER 推断 domain。
```

所以本次不是删除 NER，而是：

```text
保留 NER 能力。
抽离到新的 medical_ner.py。
主路径直接依赖 MedicalNER。
旧 medical_standardizer.py 暂时保留，后续再归档或删除。
```

---

### 3. 新增文件

新增：

```text
backend/services/medical_ner.py
```

职责：

```text
1. 加载 Clinical-AI-Apollo/Medical-NER。
2. 提供 extract_entities(text)。
3. 提供 is_medical(text)。
4. 提供 infer_domain(text)，把 NER label 映射成检索 domain。
```

核心语义：

```text
fallback candidate expansion
  -> MedicalNER.infer_domain(expansion)
  -> domain
  -> Drug 走 RxNorm，其它医学概念走 SNOMED
```

---

### 4. 修改文件

修改：

```text
backend/services/abbr_service.py
```

改动前：

```python
from services.medical_standardizer import MedicalStandardizer

self.standardizer = MedicalStandardizer()

```

改动后：

```python
from services.medical_ner import MedicalNER

self.ner_service = MedicalNER()
```

fallback domain 推断也从：

```python
_, label, _ = self.ner_service.is_medical(candidate.get("expansion"))
candidate["domain"] = NER_LABEL_TO_DOMAIN.get(label)
```

改为：

```python
domain, _, _ = self.ner_service.infer_domain(candidate.get("expansion"))
candidate["domain"] = domain
```

## 2026-07-10 Primary 缩写词典手动编辑后热刷新修复

### 问题现象

手动删除 `backend/data/abbr_candidates.py` 中新增的 `RA` 候选后，Analyze 页面重新分析：

```text
The patient has RA with joint pain and morning stiffness.
```

仍然可能显示 RA 来自 `primary`。

### 根因

`ABBR_CANDIDATES` 是 Python 模块导入时加载到内存中的 dict。写入按钮会同步更新内存，但用户手动编辑 `abbr_candidates.py` 后，如果后端进程没有真正退出，内存中的旧候选仍然存在。

这会造成：

```text
文件里 RA 已删除
但运行中的后端内存里 RA 仍存在
```

### 修复方式

修改文件：

```text
backend/data/abbr_candidates.py
backend/services/abbr_candidate_retriever.py
backend/services/abbr_service.py
```

本次新增：

```text
reload_abbr_candidates_if_changed(force=False)
```

行为：

```text
1. 记录 abbr_candidates.py 的文件修改时间。
2. 每次候选召回前检查文件 mtime。
3. 如果文件发生变化，重新解析 ABBR_CANDIDATES = {...}。
4. 原地 clear/update 同一个 ABBR_CANDIDATES dict。
5. 已经 import 该 dict 的模块会自动看到最新内容。
```

调用位置：

```text
ABBRCandidateRetriever.retrieve()
ABBRService._get_abbreviation_candidates()
```

这样手动删除或修改 `abbr_candidates.py` 后，不需要依赖完整重启，也能让 primary 词典和文件保持一致。
