# 项目改进方案 · 完整版（V10 重构：详细流程图 + mapping 级颗粒度）

> 这是把"逐模块深挖"的全部结论收口成的一份**完整改进设想**。**不是现有代码,是重构方案**——面试时作为"我读懂了 V9、能批判它、并能重构"的弹药。
> 本篇相对模块21 的升级:① 流程图每一阶段的细节补全;② 把 **mapping 级颗粒度重写 / 失败隔离** 真正画进流程,并单列一张 **per-mapping 状态机**;③ 修正早停哨兵措辞;④ 补全循环边界、reflect 两种模式、单候选弃权、继续条件。

---

## 〇、一句话总纲

> **扩写的本质是"把缩写换成选定的全称"。** V9 用 LLM 造句来做这件事,反而引入"丢否定 / 幻觉"风险——而这正是它后面要费力 verify 补救的。重构的主线就是:**让 coverage 选出唯一最佳扩写 → 确定性替换 → 把工作单位降到单个 mapping → 失败只隔离重修它自己。** 指导思想四个字:**确定性优先**。

五处核心改动(详见第四节对照表):

1. NER 从"整句标准化"挪到 **fallback 候选校验 + 产出类型 label**;
2. coverage 直接选出 **唯一最佳 expansion**(不再只返回 plausible 列表);
3. 扩写 LLM 换成 **确定性 token 边界替换**;
4. SNOMED 检索用 NER 的 label **软约束 domain**(激活那个一直传 None 的死参数);
5. verify 瘦身(句子级可省)+ reflect 按 **mapping 粒度** 只修出错项 + **增量重算**。

## 二、mapping 级状态机（本方案的核心，单列一张）★

V9 的工作单位是"整句",一个词错 → 整句重写、连累对的。重构把工作单位降到**单个 mapping**,每个 mapping 走自己的状态链,**失败只隔离它自己**。

### 2.1 单个 mapping 的生命周期

```text
            ┌─────────────────────────────────────────────┐
            │  召回阶段（循环外）                          │
            │   coverage 给这个缩写选唯一最佳 expansion     │
            └───────────────┬─────────────────────────────┘
                  选出 │              │ 选不出（无候选/全否）
                       ▼              ▼
                  status=PENDING   不产生 mapping 条目
                  （已替换进句子）  （→ 计入早停判断）
                       │
        ┌──────────────┼────────────────── 进入重试循环 ───────────────┐
        │              ▼                                                │
        │       ┌─ 检索 SNOMED（按【被选中的】expansion，非候选池）─┐  │
        │       │  domain 软加分；增量：变过才重检索               │  │
        │       └──────────────────┬───────────────────────────────┘  │
        │                          ▼                                   │
        │                   verify（映射级）                           │
        │              通过 │            │ 不通过                       │
        │                   ▼            ▼                             │
        │            ╔═══════════╗   还有未试候选?                     │
        │            ║ LOCKED_OK ║    │有          │无                 │
        │            ║  (冻结)   ║    ▼            ▼                   │
        │            ╚═══════════╝  reflect 换选  ╔═══════════════╗    │
        │                 ▲         一个新候选     ║ LOCKED_ABSTAIN ║    │
        │                 │         expansion变了  ║   (弃权不扩)   ║    │
        │                 │         → 回到检索      ╚═══════════════╝    │
        │                 └──────────────┘                            │
        └──────────────────────────────────────────────────────────────┘

  三个终态：
    LOCKED_OK        —— 扩写成功 + 有 SNOMED 支持，冻结，进 standardized_entities
    LOCKED_ABSTAIN   —— 候选都试遍了仍不过 → 不扩（保留原缩写/送人工），安全失败
    （NO_EXPANSION）  —— 召回阶段就没选出，根本没进 mappings，只影响早停
```

### 2.2 为什么这套能"只重修出错的、锁定对的"——三个前提缺一不可

```text
前提① 状态分阶段锁：每个 mapping 有自己的 PENDING→检索→校验→LOCKED 链，
        且能区分失败模式（选错 vs SNOMED 缺概念），不再是笼统一个 is_valid。

前提② 工作单位 = 单个 mapping：reflect 只碰 FAIL 的那条，LOCKED_OK 的冻结复用。

前提③ 确定性替换（使能条件，最关键）★：
        只有"局部 token 替换"才物理隔离——改 MS 那个位置不可能影响 CP 那个位置。
        若仍用 LLM 整句重写，它一开口就是整句，无法只动 MS 不碰 CP → 隔离失效。
        ∴ 第③条（确定性替换）和本节（mapping 粒度）是【绑死】的同一个决策的两面。
```

### 2.3 跑例对比（CP 对、MS 错）

```text
输入 "Patient has CP and MS"

【V9 整句重来】
 轮1: "...chest pain...multiple sclerosis" → MS错 → all()→整轮fail
 轮2: LLM 重写整句 → 有概率连 chest pain 也动；即使没动也白白重检索 CP
      成本：2次整句检索 + 2次verify + 1次整句LLM重写
      风险：本来对的 CP 被牵连改坏

【重构 mapping 粒度】
 召回: CP池={chest pain}; MS池={multiple sclerosis, mitral stenosis}，初选 multiple sclerosis
 轮1: CP→chest pain  检索+verify 通过 → LOCKED_OK（冻结）
      MS→multiple sclerosis 检索+verify 失败（有杂音=心脏病）→ 有未试候选 → reflect
      reflect 换选 mitral stenosis（候选池没变，选择变了）
 轮2: CP 已 LOCKED_OK → 跳过，复用缓存          ← CP 完全没被惊动
      MS→mitral stenosis 重检索（输入变了，SNOMED 命中心脏瓣膜）+verify 通过 → LOCKED_OK
      全部锁定 → 退出
      成本：CP 1次；MS 2次。无整句重写，无牵连。
```

## 三、两个最易被追问的边界（循环边界 + reflect 模式）

### 3.1 循环边界：召回在外、检索+校验+reflect 在内

判据:**循环只圈"重试能改变结果"的步骤。**

```text
循环外（算一次）：gate → 词典 → fallback(top-k) → NER → coverage
    = 建【候选池】+ 初选。理由：
      · gate 确定性，重跑等于复读；
      · 词典/fallback/NER/coverage 昂贵且结果基本不变（候选池不随重试变）。

循环内（每轮跑）：检索 → verify → reflect(换选/弃权)
    = 探索"从池里选哪个"。理由：
      · reflect 会改【被选中的 expansion】；
      · 检索的输入是【被选中的那个词】，不是候选池——
        ★池固定 ≠ 检索输入固定。reflect 一换词，检索输入就变，必须重检索。
```

关键澄清(你最容易绕进去的点):**检索查的是"被选中的 expansion",不是"候选池"。** 池由召回定死(所以召回在循环外);但"选池里哪个"在变(reflect 在循环内);检索跟着"选哪个"走,所以检索也在循环内。轮2 检索 `mitral stenosis` 绝非复读轮1 的 `multiple sclerosis`——查的词不同,SNOMED 结果全新。

### 3.2 reflect 的两种模式 + 单候选 → 弃权

"reflect 换候选"默认池里有 ≥2 个候选。**单候选**(单义词典 / fallback 只产一个)时无目标可换。解决:

```text
模式A 选择型 reflect（本方案主用）：池里有未试候选 → 改选一个。
      ← 所以 fallback 改为一次产 top-k（≥3），从源头保证有可换空间。

模式B 生成型 reflect（不推荐主用）：靠 LLM 带 verify 报告重新【生成】一个。
      ← 又把 LLM 造词请回来，违背"确定性优先"，仅作极端回退。

边界统一：单候选不是 bug，是"候选用尽→弃权"的最小情形（N=1）。
   某 mapping 的【未试候选】用尽且仍不过 verify → LOCKED_ABSTAIN（不扩）。
   这正是"不该扩就不扩"的安全失败：医疗场景宁可不扩，也不乱扩。
```

**继续条件也随之精确化**:循环要不要再转,**判据是"还有没有失败 mapping 手里有未试候选",不是单纯的轮次**。全部 mapping 要么 LOCKED_OK、要么 LOCKED_ABSTAIN,就退出;`max_retries` 只作防极端的兜底上限。

## 四、五处改动逐条对照 V9（为什么改）

| #                | V9 现状                                  | 重构版                                                       | 为什么                                                       |
| ---------------- | ---------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| ① NER 位置       | 在 `standardize(整句)`,冗余、不喂 verify | 挪到 **fallback 候选校验**(is_medical)+**产 label**          | 让 NER 承重、免整句合并问题、跨模型缓解同源盲区              |
| ② coverage 输出  | 返回 plausible **列表**,扩写 LLM 再选    | 直接选出**唯一最佳** expansion                               | 消歧本就该 coverage 拍板;选唯一才能确定性替换                |
| ③ 扩写           | **LLM 造句**(引入丢否定/幻觉)            | **确定性 token 边界替换**                                    | 既已选定,替换更简单/便宜/安全;从源头消除否定保持问题;**且是 mapping 粒度隔离的使能条件** |
| ④ 检索 domain    | `domain_filter` 一直传 None(死参数)      | 用 NER 的 label **软加分**约束 domain                        | 激活死参数+提精度;软加分避免映射错时弄丢答案                 |
| ⑤ verify/reflect | 句子级+映射级双查;整句重写;all 偏严      | 句子级**可省**;reflect 按 **mapping 粒度**换候选/弃权;**增量**重算 | 替换不改坏句子→句子级没必要;只修出错项→省成本、不误伤对的    |

附带激活/补强:fallback 产 top-k(配合②⑤的可换空间);失败 mapping 走 ABSTAIN(配合"不该扩就不扩")。
